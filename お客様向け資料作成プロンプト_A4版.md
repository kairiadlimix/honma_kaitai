# お客様向け資料作成プロンプト（A4 1枚版）

## 資料の目的
本間解体工業様向け「重機稼働予測ダッシュボード」プロジェクトの提案資料をA4 1枚で作成する。

---

## 資料構成（A4 1枚に収める）

### 1. プロジェクト概要（3-4行）
- **背景**: 重機の故障率を定量的に把握、買い替え時期の推測、コスト見える化
- **目的**: 各重機の故障率算出、買い替え時期推測、AI/MLモデル構築
- **期待効果**: コスト見える化、故障予防、買い替えタイミング最適化

### 2. システム構成（簡潔に）
- **技術**: Next.js 14 + TypeScript + PostgreSQL
- **データソース**: kintone（本番） / Google Sheets（PoC）
- **デプロイ**: AWS / Azure（既存環境活用）

### 3. 機能一覧（箇条書き、簡潔に）

**✅ 実装済み**
- 全社サマリー（KPI、アラート）
- 重機一覧（テーブル）
- 稼働ランキング（複数ランキング）
- 消耗品管理（グラフ、ランキング）
- コスト分析（グラフ、ランキング）

**⏳ 実装予定（PoC完成まで）**
- 故障リスク分析
- 重機個別ページ

**🔒 本番環境で追加**
- 認証・認可（ロール別アクセス制御）
- kintone連携
- 自動更新機能

### 4. 開発スケジュール（表形式、簡潔に）

| フェーズ | 期間 | 内容 | 工数 |
|---------|------|------|------|
| **フェーズ1: PoC完成** | 3-4週間 | 全機能実装、Google Sheets連携 | 15-20日 |
| **フェーズ2: 本番準備** | 4-5週間 | 認証・認可、kintone連携、テスト | 20-25日 |
| **フェーズ3: 予測機能** | 6-8週間 | AI/MLモデル開発、統合 | 28-40日 |

※ 工数にはバッファ（安全マージン）を含む

### 5. 認証・認可（簡潔に）

- **認証**: NextAuth.js（OAuth、JWT）
- **ロール**: 経営層（フルアクセス）、企画/管理（集計値）、整備部署（詳細データ）、現場担当（担当現場のみ）

### 6. 成果物
- ダッシュボードアプリケーション
- ソースコード、技術ドキュメント
- ユーザーマニュアル

### 7. 次のステップ
1. PoC完成（故障リスク分析、重機個別ページ）
2. データ連携（Google Sheets → kintone）
3. 本番環境への移行

## A4 1枚に収めるためのコツ

### 1. 情報の優先順位
- **必須**: プロジェクト概要、スケジュール、機能一覧
- **重要**: 認証・認可、成果物
- **省略可能**: 詳細な技術仕様、リスク詳細

### 2. レイアウト
- **2カラム構成**: 左側に概要、右側にスケジュール
- **表を活用**: スケジュール、機能一覧は表形式
- **箇条書き**: 詳細説明は箇条書きで簡潔に
- **フォントサイズ**: 10-11pt程度で読みやすく

### 3. 視覚的な工夫
- **色分け**: 実装済み（緑）、実装予定（黄）、本番追加（青）
- **アイコン**: 各セクションにアイコンを配置
- **余白**: 適切な余白で読みやすく

### 4. 省略する内容
- 詳細な技術仕様（別資料で）
- リスクの詳細説明（簡潔に）
- 画面イメージ（別資料で）
- 詳細な工数内訳（合計のみ）
