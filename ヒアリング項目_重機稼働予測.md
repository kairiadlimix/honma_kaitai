## 本間解体工業様向け ヒアリングシート
### テーマ：重機稼働・コスト・故障のダッシュボード / 予測設計

> このシートは「これだけ決まればダッシュボードと分析基盤が設計できる」という観点で項目を整理しています。

---

### 1. ビジネスゴールとユースケース
- **1-1 ゴール定義**  
  このプロジェクトで、経営として一番達成したいことは何ですか？
  - 例：重機コストの見える化 / 故障予防 / 配車・稼働効率の最大化 など

- **1-2 主要ユースケース（誰が・いつ・何を見るか）**  
  代表的な「シーン」を3つ程度挙げてください。  
  例：
  - 経営層：月次で重機コストと稼働状況を俯瞰したい
  - 工事部長：週次で現場別の重機稼働とトラブルを把握したい
  - メンテ担当：日次で故障リスクの高い重機を把握したい

---

### 2. ダッシュボード構成イメージ
- **2-1 作りたい画面の単位**  
  どのような「ビュー（画面）」があると便利そうですか？
  - 例：
    - 全社サマリー（全重機・全現場）
    - 現場別ビュー
    - 重機個別ビュー

- **2-2 それぞれの画面で“必ず置きたい”指標・グラフ**  
  画面ごとに、最低限必要な指標・グラフを挙げてください。  
  例：
  - 全社サマリー：総稼働時間、総コスト、故障件数、ワースト/ベスト重機ランキング
  - 現場別：現場ごとの稼働率、重機台数、トラブル件数
  - 重機別：月次稼働時間推移、故障履歴、メンテ履歴、年間コスト

- **2-3 時系列の粒度**  
  時系列はどの粒度で見たいですか？（日 / 週 / 月 / 四半期 など）

---

### 3. KPI・指標設計
- **3-1 主要KPI（3〜5個）**  
  「これが一目で分かれば意思決定しやすい」というKPIを挙げてください。  
  例：
  - 重機稼働率
  - 重機1台あたり年間コスト
  - 故障件数 / 故障発生率
  - メンテ計画遵守率

- **3-2 比較したい軸**  
  KPIをどの軸で比較できるとよいですか？  
  例：重機種別 / 現場 / 時期 / メーカー / 年式 など

- **3-3 アラート条件**  
  「この条件を超えたらアラートを出したい」という閾値や状態はありますか？  
  例：
  - 故障回数が◯回/月を超えた重機
  - 稼働率が◯％未満の重機

---

### 4. 分析に使う“単位（エンティティ）”と粒度
- **4-1 主な分析単位**  
  分析・表示の基本単位として、次のうちどれを主に使いますか？（複数可）
  - 重機個体（ID単位）
  - 重機種別（型式・容量など）
  - 現場（案件）
  - 期間（年月 / 週 / 日）

- **4-2 1レコードのイメージ**  
  「1行が何を表すか」のイメージを教えてください。  
  例：
  - 1行 = 「重機×日」の稼働実績
  - 1行 = 「故障イベント」
  - 1行 = 「メンテナンス作業」

---

### 5. データソースと更新頻度
- **5-1 使いたいデータソース**  
  利用を想定しているシステム・データの種類を挙げてください。  
  例：
  - 重機メンテナンスアプリ
  - kintone
  - 現場日報（Excel / 紙）
  - 会計・原価システム など

- **5-2 更新頻度のイメージ**  
  ダッシュボードはどのくらいの頻度で更新されていれば十分ですか？  
  例：リアルタイム / 1日1回 / 週1回

- **5-3 データ取得方法の前提**  
  API / CSV / 手動アップロード など、想定している取得方法があれば教えてください。

### 6. 予測（将来拡張）の前提
- **6-1 予測したいテーマの優先順位**  
  将来的に、どのような予測ができると一番価値がありますか？
  - 故障リスク予測
  - 稼働時間・稼働率の予測
  - 来期の重機コスト予測 など

- **6-2 予測結果の使い方**  
  予測結果を、どの画面・どの意思決定に組み込みたいですか？

### 7. 権限・閲覧制御

  経営層 / 部長 / 現場所長 / メンテ担当 など、それぞれどこまで見られるべきでしょうか？

  対外説明や、将来の他社展開を見据えて、どのデータを匿名化・集計レベルに留める必要がありますか？
