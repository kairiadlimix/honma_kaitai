# デモモード起動方法（サンプル画面）

## 🎯 目的

Google Sheetsの設定が完了していなくても、モックデータでダッシュボードを表示できます。
社内共有用のサンプル画面として使用できます。

## 🚀 起動方法

### 方法1: 環境変数でモックデータモードを有効化（推奨）

`.env.local`ファイルを作成（または既存のファイルに追加）：

```env
# モックデータモードを有効化
USE_MOCK_DATA=true
```

### 方法2: GOOGLE_SHEETS_IDを設定しない

`.env.local`ファイルに`GOOGLE_SHEETS_ID`を設定しなければ、自動的にモックデータモードになります。

## 📝 手順

### 1. .env.localファイルを作成

```bash
cd /Users/kairi.oshima/honma_kaitai/dashboard
touch .env.local
```

### 2. モックデータモードを有効化

`.env.local`ファイルに以下を記述：

```env
USE_MOCK_DATA=true
```

### 3. 開発サーバーを起動

```bash
npm run dev
```

### 4. ブラウザで確認

`http://localhost:3000` を開く

## ✅ 表示される内容

- **総重機台数**: 15台
- **稼働中**: 12台
- **整備中**: 2台
- **入庫中**: 1台
- **全体稼働率**: 65.5%
- **今月の総コスト**: 4,850,000円
- **故障リスクが高い重機**: 3台
- **特自検期限が近い重機**: 2台
- **稼働率が低い重機**: 5台

## 🔄 実際のデータに切り替える方法

Google Sheetsの設定が完了したら：

1. `.env.local`から`USE_MOCK_DATA=true`を削除
2. `GOOGLE_SHEETS_ID`を設定
3. サービスアカウント情報を設定
4. 開発サーバーを再起動

## 📊 デモ用のデータ

モックデータは以下の内容です：
- 15台の重機（コベルコ8台、日立7台）
- 様々なステータス（稼働中、整備中、入庫中）
- リアルな数値（稼働率、コストなど）

## 💡 使い方

1. **社内共有**: デモモードで起動して、ダッシュボードの見た目と機能を確認
2. **プレゼン**: 実際のデータがなくても、システムのイメージを共有可能
3. **開発**: Google Sheetsの設定を待たずに、フロントエンドの開発を進められる
